SharePoint implementation notes
  • Use a SharePoint communication site with a custom login/landing page (SPFx full-page application or custom web part)

  • Footer links (Privacy, Terms, Cookies) open external URLs or document library PDFs

SharePoint implementation notes
  • Build with Microsoft Power Apps embedded form or Microsoft Forms iframe on a SharePoint page
  • Submissions write to a SharePoint list ("Access Requests") with columns matching each field
  • Power Automate flow triggers on new list item: emails the marketing/DAM admin with approve/deny action links
  • "Internal sponsor" field helps verify the relationship — consider a people picker if Azure AD lookup is available
  • On approval: Power Automate adds user to Azure AD B2B guest group with scoped permissions to external document libraries
  • On denial: automated email to requester with optional reason
  • Terms checkbox required before submission; terms PDF stored in a public document library

SharePoint implementation notes
  • External users land on a SharePoint communication site with audience-targeted navigation (hide internal-only nav items)
  • Asset categories are document library views filtered by a "Category" managed metadata column
  • Permission-trimmed security: external Azure AD guest group has read-only access to specific libraries/folders

  • "Recent assets" section: Highlighted Content web part filtered to latest published items
  • Download buttons link directly to file URLs; track downloads via Power Automate HTTP trigger
  • External users should NOT see: upload buttons, draft assets, admin tools, user management, or analytics

SharePoint implementation notes
  • Internal home is a SharePoint communication site home page with custom web parts
  • Stats row: embedded Power BI tiles or custom SPFx web parts querying SharePoint REST API for counts
  • Asset library: document library with managed metadata columns (Type, Category, Status) in a custom gallery view
  • Filters use metadata navigation or a custom SPFx filter web part
  • Pending requests: pull from "Access Requests" SharePoint list; approve/deny buttons trigger Power Automate flows
  • Activity feed: SharePoint site activity API or a custom list logging uploads/downloads/approvals via Power Automate
  • Nav tabs (Dashboard, All assets, Upload, Collections, Users, Analytics) are SharePoint site pages
  • "Collections" are document sets or metadata-grouped views for curating themed asset bundles
  • "Users" page: custom SPFx web part pulling Azure AD guest users from Microsoft Graph
  • "Analytics" page: embedded Power BI dashboard — download trends, popular assets, user engagement

SharePoint implementation notes
  • Asset detail is a custom page layout driven by document library item metadata
  • Preview: SharePoint's built-in file preview (Office files, images, PDFs, videos)
  • Version history: native SharePoint document versioning displayed as thumbnails
  • Tags: managed metadata term sets for consistent taxonomy
  • Visibility field: choice column ("Internal only" / "Internal + External") controlling Azure AD group access
  • "Edit metadata" opens the SharePoint list item edit panel or a custom SPFx form
  • "Archive" moves asset to an archive folder / sets status to "Archived" (soft delete)
  • Permission-trim action buttons: hide "Edit metadata" and "Archive" for external users via audience targeting or SPFx conditional rendering
  • Download tracking: log downloads to a SharePoint list via Power Automate for analytics

SharePoint implementation notes
  • Upload form maps to SharePoint document library columns: Title, Description, Category (managed metadata), Visibility (choice), Tags (multi-value managed metadata)
  • Drag-and-drop: native SharePoint file upload or a custom SPFx upload web part
  • File type restrictions enforced at the library level (blocked file types setting)
  • 100 MB limit: configurable in SharePoint admin; default tenant limit is 250 GB per file
  • "Save as draft" sets content approval status to "Pending" — visible internally but not to externals
  • "Publish immediately" sets content approval to "Approved" — may trigger approval flow if enabled
  • Consider requiring content approval on the library so all uploads are reviewed before external visibility
  • After upload, redirect to the Asset Detail page (page 5) for the newly created item
