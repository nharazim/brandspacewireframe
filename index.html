SharePoint implementation notes
  • Use a SharePoint communication site with a custom login/landing page (SPFx full-page application or custom web part)
  • "Internal user" button redirects to Azure AD SSO — on success, route to Internal Home (page 4)
  • "External user" button redirects to Azure AD B2B guest login — on success, route to External Home (page 3)
  • "Request vendor access" links to the Request Access form page (page 2)
  • Left panel: use a full-bleed hero image or CSS background. Consider rotating seasonal brand imagery.
  • Footer links (Privacy, Terms, Cookies) open external URLs or document library PDFs

SharePoint implementation notes
  • Build with Microsoft Power Apps embedded form or Microsoft Forms iframe on a SharePoint page
  • Submissions write to a SharePoint list ("Access Requests") with columns matching each field
  • Power Automate flow triggers on new list item: emails the marketing team / DAM admin with approve/deny links
  • "Internal sponsor" field helps admins verify the relationship — consider a people picker if Azure AD lookup is available
  • On approval: Power Automate adds the user to an Azure AD B2B guest group with scoped permissions to the external document libraries
  • On denial: automated email to the requester with optional reason
  • Terms checkbox is required before submission; terms PDF stored in a public document library

SharePoint implementation notes
  • External users land on a SharePoint communication site with audience-targeted navigation (hide internal-only nav items)
  • Asset categories are SharePoint document library views filtered by a "Category" managed metadata column
  • Use permission-trimmed security: external Azure AD guest group has read-only access to specific libraries/folders
  • Search bar uses SharePoint search scoped to the DAM site with a custom search results page
  • "Recent assets" section: use a Highlighted Content web part filtered to the user's recent activity or latest published items
  • Download buttons link directly to file URLs; consider tracking downloads via a Power Automate HTTP trigger
  • External users should NOT see: upload buttons, draft assets, admin tools, user management, or analytics

SharePoint implementation notes
  • Internal home is a SharePoint communication site home page with custom web parts
  • Stats row: use embedded Power BI tiles or custom SPFx web parts querying SharePoint REST API for counts
  • Asset library: SharePoint document library with managed metadata columns (Type, Category, Status) displayed in a custom gallery view
  • Filters use metadata navigation or a custom SPFx filter web part
  • Pending access requests: pull from the "Access Requests" SharePoint list (page 2); approve/deny buttons trigger Power Automate flows
  • Activity feed: use SharePoint site activity API, or a custom list that logs uploads/downloads/approvals via Power Automate
  • Nav tabs (Dashboard, All assets, Upload, Collections, Users, Analytics) are SharePoint site pages
  • "Collections" are SharePoint document sets or metadata-grouped views for curating themed asset bundles
  • "Users" page lists Azure AD guest users with their access status — consider a custom SPFx web part pulling from Microsoft Graph
  • "Analytics" page: embedded Power BI dashboard showing download trends, popular assets, user engagement

SharePoint implementation notes
  • Asset detail is a custom SharePoint page layout driven by document library item metadata
  • Preview area: use SharePoint's built-in file preview (supports Office files, images, PDFs, videos)
  • Version history: leverage native SharePoint document versioning — display major versions as thumbnails
  • Tags: use managed metadata term sets for consistent taxonomy; display as pills
  • Visibility field: a choice column ("Internal only" / "Internal + External") that determines which Azure AD groups can see the item
  • "Edit metadata" opens the SharePoint list item edit panel (or a custom SPFx form)
  • "Archive" moves the asset to an archive folder / sets status to "Archived" (soft delete, not permanent)
  • Permission-trim action buttons: external users should not see "Edit metadata" or "Archive" — use audience targeting or conditional rendering in SPFx
  • Download tracking: log downloads to a SharePoint list via Power Automate (triggered by file access) for analytics

SharePoint implementation notes
  • Upload form maps directly to SharePoint document library columns: Title, Description (multi-line text), Category (managed metadata), Visibility (choice), Tags (multi-value managed metadata)
  • Drag-and-drop: use native SharePoint file upload or a custom SPFx upload web part for a richer experience
  • Supported file types should be enforced at the library level (blocked file types setting)
  • 100 MB limit: configurable in SharePoint admin; default tenant limit is 250 GB per file
  • "Save as draft" sets the content approval status to "Pending" — asset is visible internally but not to external users
  • "Publish immediately" sets content approval status to "Approved" — if approval workflows are enabled, this triggers an approval flow first
  • Consider enabling content approval on the document library so all uploads go through a review cycle before external visibility
  • After upload, redirect to the Asset Detail page (page 5) for the newly created item
